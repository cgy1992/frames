Detailed Description

Coordinator class for almost all Frames state. Every Frames-using program will contain at least one Environment.

A Frames::Environment is a self-contained Frames instance. It handles input, owns a hierarchy of Frames, manages resources, and contains all the configuration data needed to use Frames.

While Environment is not locked to any individual thread, it is fundamentally single-threaded. It is undefined behavior to call any Environment function, or any function on a Frame owned by an Environment, while any other such function is being run in another thread. However, multiple Environments can be used in parallel without issue.

Except when specified otherwise, it is always valid to call an Environment or Frame function while handling an event dispatch in that same thread.

Environment will clean itself up entirely when it is destroyed, including destroying all child Frames, cleaning up all stored resources and event handles, cleaning up any hooks in scripting languages, and so forth.

bool Frames::Environment::Input_MouseDown ( int  button)

Informs the environment that a mouse button has been pressed.

Button state changes happen immediately and may trigger Layout::Event::Mouse*Down events. All integers are valid and will trigger the appropriate Layout::Event::MouseButton* event. However, 0 will also trigger Layout::Event::MouseLeft* events, 1 will also trigger Layout::Event::MouseRight* events, and 2 will also trigger Layout::Event::MouseMiddle* events.

Returns true if the event was consumed by a Frame.
